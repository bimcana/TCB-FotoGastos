### Task 2 — Metadatos por archivo + conciliación (TDD)
- `src/indice.js`: `descDeEntrada(entrada) → string JSON {v:1,...}` y `entradaDeDesc(str) → entrada|null` (tolerante a basura); `conciliarIndice(idx, archivos)` puro: `archivos=[{name,description}]` → `{indice, restauradas:[entrada], sinProcesar:[name]}` — restaura entradas con description válida ausentes del índice (re-marcando duplicada por NCF contra el índice resultante), clasifica "Sin procesar" el resto de imágenes no indexadas. Tests de ida-vuelta, basura, restauración y dup.
- `src/drive.js`: scope → `https://www.googleapis.com/auth/drive`; `listarArchivos(carpetaId) → [{id,name,mimeType,description}]`; `listarCarpetas(padreId) → [{id,name}]`; `descargarPorId(fileId) → Blob`; `moverAPapelera(fileId)` (PATCH trashed:true); `subirJPEG(blob,nombre,carpetaId,description?)`; `ponerDescripcion(fileId, texto)`; `moverYRenombrar(..., description?)`; fix escape backslash en `buscarCarpeta` (como `asegurarCarpeta`).
- `src/main.js`: `subirFactura` escribe description al subir; `actualizarEntradaConReArchivo` re-escribe description en cada cambio (con o sin re-archivo).

### Task 3 — Gastos acordeón
`index.html`: la tarjeta del mes se vuelve contenedor `#gastos-arbol`; fuera el selector ‹ › (y su CSS queda sin uso: se retira). `main.js`: `refrescarGastos()` lista `listarCarpetas(raizId)` + `listarArchivos(raizId)` (sueltos) → secciones (meses desc, otras alfabético, «Carpeta principal» si hay sueltos); estado `seccionesAbiertas` (Set, mes actual inicial); `renderSeccion(carpeta)` al expandir: `listarArchivos` + `leerJSON` + `conciliarIndice` (restauradas se persisten bajo `conLockIndice`) → filas con chips actuales + "Sin procesar" (chip warn) + botón Generar al pie si es mes. `abrirRevisar(ctxCarpeta, archivo)` y `generarDocumento(ctxCarpeta)` reciben contexto explícito; muere `mesVisto`/flechas. CSS acordeón (.acc-head con chevron, .acc-body).

### Task 4 — Flujo "Sin procesar"
Fila "Sin procesar" tocable → `procesarAjena(ctxCarpeta, {id,name})`: `descargarPorId` → `archivoACanvas` (HEIC: catch → toast honesto) → `detectarDocumento(1200)` → `detectarConIA` → `abrirEditorEsquinas` SIEMPRE precargado → `window.__captura` + `window.__origenAjeno = {fileId, nombre}` → `procesarYRevisar()`. En el confirm handler, tras subir OK: si `__origenAjeno` → `moverAPapelera(fileId)` (fallo → toast «El original sigue en la carpeta; puedes reintentarlo») y limpiar flag. `entradaDeFactura` gana `origen:'importada'` + `procesadaDesde`. Al cancelar (volver a cámara) se limpia el flag sin tocar el original.

## Apéndice — Calibración con 57 facturas reales (T1, 2026-07-19)

**Detección:** clásica 36/57 (63%) · IA local rescata 12 más (48/57 = 84%) · 9 "ninguno".
Los 9 sin detección son fotos YA RECORTADAS (el recibo llena el encuadre — escaneos y
reenvíos de WhatsApp; verificado visualmente en c06/c42): ambos motores las rechazan por la
guarda anti-encuadre-completo (>98% del área), lo cual es CORRECTO. Consecuencia de diseño:
el editor de esquinas abre por defecto a MARCO COMPLETO (inset 2%, antes 10%) — para una
foto ya recortada, "Aplicar" sin mover nada es la acción correcta. Cobertura efectiva ≈ total.

**Ratios alto/ancho post-recorte (57):** continuo de 1.08 a 3.80 (cartas 1.1–1.3,
restaurantes/farmacias 1.6–2.9, gasolineras/tickets medianos 3.0–3.8) → **HUECO** →
4.35 (c46) y 5.95 (c02 = supermercado Sirena). **Umbral calibrado: `RATIO_LARGA = 4`** —
solo los 2 tickets de supermercado se dividen; ninguna gasolinera/restaurante se divide
(regla de altura de Ari). c42 es un caso curioso (2 vouchers en una imagen) que el usuario
resuelve recortando a uno en el editor.
